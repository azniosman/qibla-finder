# Security Policy

## Overview

The security and privacy of Qibla Finder users is our top priority. This document outlines our security policy and how to report security vulnerabilities.

## Supported Versions

We currently support the following versions with security updates:

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < Latest| :x:                |

We recommend always using the latest version of Qibla Finder to ensure you have the most recent security updates.

## Security Features

- **Local Data Storage**: All data is stored locally on your device using AsyncStorage
- **No External Servers**: Zero data transmission to third-party servers
- **Location Privacy**: GPS location is used only for prayer time calculations and is not stored or transmitted
- **Minimal Permissions**: Only essential permissions (Location, Notifications) are requested
- **Open Source**: Full transparency through open source code

## Reporting a Vulnerability

If you discover a security vulnerability in Qibla Finder, please help us protect our users by reporting it responsibly.

### How to Report

1. **DO NOT** create a public GitHub issue for security vulnerabilities
2. Email the security concern to: [azniosman@users.noreply.github.com]
3. Include the following information:
   - Type of vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### What to Expect

- **Acknowledgment**: We will acknowledge receipt of your vulnerability report within 48 hours
- **Updates**: We will provide regular updates on our progress (at least every 7 days)
- **Resolution Timeline**: We aim to resolve critical vulnerabilities within 30 days
- **Credit**: With your permission, we will credit you in the security advisory and release notes

### Vulnerability Assessment

**Accepted Vulnerabilities:**
- We will work with you to understand and resolve the issue
- A fix will be developed and tested
- A security advisory will be published
- Credit will be given to the reporter (if desired)

**Declined Vulnerabilities:**
- We will provide a clear explanation of why the issue does not constitute a security vulnerability
- Alternative solutions or mitigations may be suggested

## Security Best Practices for Users

1. **Keep Updated**: Always use the latest version of Qibla Finder
2. **Review Permissions**: Understand why each permission is needed
3. **Device Security**: Ensure your device has a lock screen and is kept secure
4. **App Store Downloads**: Only download from official app stores
5. **Report Issues**: If you notice suspicious behavior, report it immediately

## Data Privacy

- **No Analytics**: We do not collect usage analytics
- **No Tracking**: We do not track user behavior
- **No Account Required**: No user account or registration needed
- **Offline First**: Full functionality without internet connection
- **Data Control**: Users can export or delete all data at any time
